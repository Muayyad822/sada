# Sada Hackathon Pivot Plan: Utility App to Emotional Companion

## Executive Summary

This plan transforms Sada from a keyword-based Quran lookup into an **Emotional Companion** that creates engagement loops for sustained spiritual growth. The pivot targets **30pt Impact**, **15pt Innovation**, and **20pt UX** scores through features that keep users returning daily.

---

## Phase 1: Impact on Quran Engagement (30 pts)

### 1.1 Guided Reflection (Deepening Prompt)
**Priority: HIGH**

Add AI-generated reflective questions after each verse that connect the user's emotional state to the verse's meaning.

**Implementation:**
- Extend `mcpService.getDeepeningQuestion()` to include emotional context from the user's initial feeling
- Display question in `AudioPlayer.tsx` after each verse completes
- Create a new component `DeepeningPrompt.tsx` in `src/components/`

**Files to modify:**
- `src/components/AudioEngine/AudioPlayer.tsx` - Add prompt display after verse completion
- `src/services/mcpService.ts` - Enhance `getDeepeningQuestion()` with emotional prompts

**Example prompt for "stressed" user:**
> "In this verse, Allah mentions 'tranquility' (Sakinah) descending on hearts. What is one noise in your life you can silence tonight to let that tranquility in?"

---

### 1.2 Spaced Repetition for Heart-Verses
**Priority: HIGH**

Implement the "Heart" feature using the Quran Foundation Collection API to track favorite verses and send reminders.

**Implementation:**
- Add "Heart" button to `AudioPlayer.tsx` for saving verses to collection
- Use QF Collection API (`/api/v4/collections`) to save/bookmark verses
- Create local notification system for spaced repetition (3-day intervals)
- Store "hearted" verses in localStorage with timestamps

**API Integration:**
```
POST /api/v4/collections/{collection_id}/verses
{ "verse_key": "2:255" }
```

**Files to create/modify:**
- `src/components/AudioEngine/AudioPlayer.tsx` - Add Heart button
- `src/services/collectionService.ts` - New service for Collection API
- `src/components/SpacedRepetition/Reminder.tsx` - New component

---

### 1.3 Emotional Cycle Khatm (Badge System)
**Priority: MEDIUM**

Track which emotional themes the user has explored and award badges for completing "Emotional Cycles."

**Implementation:**
- Extend `userService` to track emotional themes in `STORAGE_KEYS.THEMES`
- Define emotion categories: Gratitude, Patience, Reliance, Trust, Hope, Sakinah
- Create badge criteria in `Dashboard.tsx`
- Display "Emotional Khatm Progress" in Stats page

**Emotion to Theme Mapping:**
- Grateful → Shukr (Gratitude)
- Anxious/Stressed → Sakinah (Tranquility)
- Lost → Hidayah (Guidance)
- Tired → Tawakkul (Reliance)
- Disappointed → Sabr (Patience)
- Lonely → Rahmah (Divine Mercy)

**Files to modify:**
- `src/components/HabitDashboard/Dashboard.tsx` - Add badge display
- `src/services/userService.ts` - Add emotion tracking
- `src/pages/Stats.tsx` - Display Khatm progress

## Phase 3: Innovation & Creativity (15 pts)

### 3.1 Contextual Mirror (Asbab al-Nuzul)
**Priority: HIGH**

Use Tafsir API to show the historical context (Asbab al-Nuzul) when a verse matches the user's emotional state.

**Implementation:**
- Enhance `mcpService.semanticSearch()` to fetch Asbab al-Nuzul
- Display "This verse was revealed when..." context in `AudioPlayer.tsx`
- Create mirroring prompts: "The Prophet felt rejected in Ta'if. How does this relate to your feeling?"

**API Integration:**
```
GET /api/v4/tafsirs/169/by_ayah/2:286
```

**Files to modify:**
- `src/services/mcpService.ts` - Add Asbab al-Nuzul fetching
- `src/components/AudioEngine/AudioPlayer.tsx` - Display contextual mirror

---

### 3.2 Voice-to-Text Reflections with Second Verse
**Priority: HIGH**

Allow users to speak their reflection, transcribe it, and receive a second verse that answers.

**Implementation:**
- Already have voice recording in `Journal.tsx`
- After user records reflection, call `semanticSearch()` with their reflection
- Display new verse with "This verse answers your reflection..."

**Flow:**
1. User records voice reflection
2. Transcribe with Web Speech API
3. Send transcript to `mcpService.semanticSearch(transcript)`
4. Display second verse that responds to their reflection

**Files to modify:**
- `src/components/TadabburJournal/Journal.tsx` - Enhance voice flow

## Phase 4: Technical Execution (35 pts)

### 4.1 Multi-Source Grounding (Translations)
**Priority: HIGH**

Fetch 2-3 translations and synthesize a "Simple English" version.

**Implementation:**


**Files to modify:**
